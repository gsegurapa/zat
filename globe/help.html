Global Tracker Documentation

This application is under development and will change without notice.

The global tracker is a flexible app that can display airline flights in a number of ways on a 3D globe. Note that only active flights (in the air, taking off, or landing) are displayed.

You can zoom the globe in and out in several ways:

  • Double-click anywhere on the globe to zoom in. Hold down the shift key and double-click to zoom out.
  • If you have a mouse with a scroll wheel, use that to zoom in and out.
  • On a touch device (phone or tablet), pinch to zoom in and out.
  • Use the zoomFactor parameter to set the initial zoom.

If auto mode is off (not rotate, or scroll), then you can drag the globe with your mouse to rotate it.

Show all arriving flights to Atlanta on Delta Airlines and their codeshares: ?airport=ATL&airline=DL&codeshares=true&arrDep=arr

Configuration

This app is currently configured using parameters in the URL (there is no interactive configuration tool yet).

airline=XX

Display flights for a specific airline using its two character airline code (three letter ICAO codes for major airlines should also work). If no airline is specified, then default to AS (Alaska Airlines). The airline logo is displayed in the lower right corner of the screen.

To display all flights for United Airlines (UA): ?airline=UA

codeshares=true

If codeshares=true is added, then also display all codeshare flights for this airline.

To display all flights for Delta Airlines (DL) and its codeshares: ?airline=DL&codeshares=true

airport=YYY

Display flights departing or arriving a specific airport using its three character airport code (four letter ICAO codes for major airports should also work). The airport code is displayed in the lower right corner of the screen.

To display all flights departing Seattle (SEA): ?airport=SEA

arrDep=arr

if arrDep=arr is added, then show arriving flights instead of departing ones.

To display all flights arriving Los Angeles (LAX): ?airport=LAX&arrDep=arr

airline=XX&airport=YYY

If both airline and airport are specified, show only flights that match both. The arrDep parameter can be added to specify whether departing (the default) or arriving flights are shown. Note that when the airport parameter is specified, then the codeshares parameter does not work. The airline logo is displayed in the lower right corner of the screen.

To display all Delta flights arriving Atlanta: ?airline=DL&airport=ATL&arrDep=arr

auto=mode

The auto parameter controls automatic scrolling and display of data.

auto=off

The default mode is off, which allows manual panning of the globe by dragging it. You can also display the flight information for any flight by clicking on its airplane icon.

auto=rotate

Sets the globe spinning: ?auto=rotate

auto=scroll

Centers the globe on each flight one after another, and displays the flight information at the bottom of the screen: ?auto=scroll

duration=time

For auto=scroll, the duration parameter controls how many seconds between each scroll: ?auto=scroll&duration=10

For auto=rotate the duration parameter controls the speed of rotation (in degrees per second, so smaller numbers correspond to a slower rotation): ?auto=rotate&duration=0.5

airplaneIcon=triangle

The default airplane icon looks like an airplane. Specifying airplaneIcon=triangle uses a simplified icon that looks like an arrowhead pointing in the direction the airplane is flying. This is faster to draw on some hardware.

scaleIcon=num

Scale the airplane icon. Default scale factor is 1.0. Larger numbers make the icon larger: ?scaleIcon=1.5

zoomFactor=num

Sets the initial zoom of the globe. The default zoomFactor is 1.0, which fits the globe exactly into the browser window. Larger numbers zoom in. To leave a border around the globe, zoom out a bit: ?zoomFactor=0.9

appId=ID&appKey=Key

Specify the FlightStats application ID and application key. Note that you must specifically request a key that can access Fleet status in order to see all flights for a carrier. During development this key is not required, but it will be required upon release.

Browser Compatibility

The app has not yet been tested extensively. It should work on recent versions of Chrome, Firefox, Safari, and on IE 9 and above. It has not been tested at all on mobile browsers. If you encounter any problems, please let us know.
