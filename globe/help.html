Configuration

The app can be configured using the interactive configuration tool. You can change any configuration parameter and see the results immediately.

Open the configuration tool by clicking on the legend below the map. In interactive mode, you can also open the configuration tool by pressing C on your keyboard. When open, you can drag the configuration tool around the map by dragging its title bar. To close the configuration tool, click on the "Hide" button, or click the Flightstats logo again.

Once you have configured the app, you can save your configuration by clicking on the "Save" button. Once saved, the app will always start up with your configuration. You can also reset the app back to the default configuration using the "Reset" button. Configurations are saved using a cookie on your local computer, so they are specific to that computer and browser.

Once you click the "Save" button and save the configuration, a URL is displayed. This URL can be used to start the app with the same configuration on a different computer or browser. Note that the appKey and appId are not saved as part of the configuration, and must be added to the startup URL.

The "Help" button brings up this help page in a new window.

The "Full" button puts the airport tracker into full screen mode, and if pushed again, back to windowed mode. Full screen mode is not supported on all browsers, so this button may not appear. When in full screen mode the mouse cursor is hidden when over the map (you can still see it over the legend).

Weather Options

The app can optionally show ground-based weather radar from the US National Weather Service (US only). To show the current weather radar, set weatherFrames to "1". Alternatively, set weatherFrames between 2 and 25 to show an animation of the recent weather. The weather frames are updated approximately every 10 minutes.

If weatherStation is set to "automatic" then the app automatically finds the closest radar station to the specified airport. You can override this by specifying a station code. See the bottom of http://www.srh.noaa.gov/jetstream/doppler/ridge_download.htm for a list of stations and their locations. This can be useful if the closest station doesn't have the best weather coverage for the airport.

Airlines List

The airlines parameter specifies a list of airline codes, separated by commas. The list is used to determine which flights to display. For example, "airlines=AA,DL" specifies that only flights operated by American Airlines and Delta Airlines should be displayed. The list can optionally begin with a single dash (minus sign), which reverses the test so that "airlines=-AA,DL" specifies that flights from all airlines should be displayed except for American and Delta.

In addition, each airline code in the list can be optionally followed by a dash and another airline code. For example, "airlines=AS,QX-AS" specifies that only flights operated by AS (Alaska Airlines), or by QX (Horizon Airlines) should be shown, but QX should be displayed as AS. Likewise, "airlines=-QX-AS" specifies that all flights should be shown, but to change QX (Horizon) to AS (Alaska). This is very useful for flights that are ticketed by one airline, but operated by a different (operating) airline. This app normally displays the operating airline, but you can use this option to display the ticketing airline. Here are some additional examples:

airlines=-Show all flights (the default)
airlines=UAShow only flights operated by United
airlines=-FX,1IShow all flights except any flights operated by FedEx or Net Jets
airlines=-OO-UA,QX-AS,MQ-AAShow all flights, but change SkyWest to United, Horizon to Alaska, and American Eagle to American
airlines=OO-UA,QX-AS,MQ-AAShow only SkyWest, Horizon, and American Eagle, but change their airline codes to
United, Alaska, and American, respectively
airlines=-JIA-USShow all flights, but change PSA to US Airways
airlines=Show no flights (not very useful)

Browser Compatibility

The app has been tested on recent versions of Chrome, Firefox, Safari, and on IE 8 and above. If you encounter any problems, please let us know.

Advanced/Experimental Options

These options must be specified in the URL, and are not available in the configurator and cannot be saved in a cookie. Some of them may not work on all platforms.

interactive=true

Setting interactive to true allows you to drag the map using your mouse pointer and to zoom in by double clicking. It also enables the following keyboard shortcuts:

+ or =zoom in
– or _zoom out
arrow keysscroll the map by 100 pixels
space barscroll back to original position (with the airport centered)
return keyreturn to original scroll and zoom
Copen/close the configuration tool
Fenter/exit full screen (if supported by the browser)
Nwhen flip=true, this advances to the next airport
When interactive is true, the mouse cursor does not hide when in full screen mode.

jump=true

Setting jump to true eliminates smooth airplane animation so that the map uses far fewer processor resources. The airplanes jump from one position to another, and the labels are turned off.

flip=true

Setting flip to true changes the airport every minute. Airports are displayed from west to east, around the world. To show only US airports, use flip=US instead. The current airport is displayed in the legend. You can also adjust the number of seconds between airport changes using the flipTime parameter. For example, flipTime=120 displays each airport for 2 minutes.

updateRate=number

By default, the app makes an API call to get positional data every 15 seconds. You can override this using the updateRate parameter. For example, to make an API call every 60 seconds, specify "updateRate=60".

The main purpose of this parameter is to reduce the cost of data calls. The downside is that the animation becomes less accurate, especially when airplanes are taking off and landing. You can experiment, but values greater than 60 seconds may not look very good.

view=3D

Setting view to 3D displays the map in psuedo-3D. Note that this does not work in all browsers. It also sets interactive mode to true, so you can drag the map around in 3D.
